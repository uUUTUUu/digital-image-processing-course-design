# HITSZ 数字图像处理课程设计

本仓库为 **哈尔滨工业大学（深圳）HITSZ《数字图像处理》课程**的课程设计项目存档，主要内容为基于 **ROS + OpenCV** 的移动机器人视觉感知与控制实现。

本项目围绕课程设计要求，完成了以下两个核心任务：

1. **锥桶道路行驶与避障**  
   小车需要沿由锥桶构成的指定道路行驶，并按照要求绕开障碍锥桶，不能抄近路。

2. **目标图案自动跟踪**  
   小车在驶出道路后，需要**自动切换**到目标跟踪模式，对印有数字图案的纸张进行识别与跟踪，过程中**不能人工切换模式**。

---

## 一、课程设计要求概述

本课程设计的任务主要分为两个阶段：

### 1. 锥桶道路行驶
- 道路由等间距摆放的锥桶构成；
- 道路中会额外放置障碍锥桶；
- 小车需要根据指定路线完成绕障和过弯；
- 必须按照规定路径通过，不能直接走捷径。

### 2. 数字目标跟踪
- 小车驶出锥桶道路后，需要自动进入目标跟踪阶段；
- 从数字目标中识别并跟踪纸张图案；
- 重点在于完成从“道路行驶”到“目标跟踪”的**自动衔接**。

---

## 二、计分要求简述

课程设计的实操评分主要按照任务完成情况和系统表现进行评定，基本要求可概括为：

- **能够识别锥桶并在道路内稳定行驶**：达到基础分；
- **能够正确避障并按指定路线通过道路**：达到更高分；
- **驶出道路后无需人工干预，自动切换到目标跟踪**：达到更高分；
- **整体运行快速、平滑、鲁棒性较好，且跟踪响应及时**：可获得更高分数。

此外，一般还会有一定的加分因素，例如：
- 使用**非深度学习方法**完成锥桶识别；
- 完成更高难度路线；
- 系统具有较好的稳定性与抗干扰能力。

因此，本项目在实现时重点关注了：
- 视觉检测的稳定性；
- 状态切换的自动性；
- 控制过程的连续性与可调性；
- 不依赖深度学习模型的传统图像处理方案。

## 三、项目文件说明

本仓库当前主要包含以下内容：

- `exp_final.cpp`  
  主程序文件，完成图像处理、状态机控制、道路行驶、障碍绕行以及数字目标跟踪等核心功能。

  ROS 工程编译配置文件。

  运行与参数配置文件，用于统一管理速度参数、转向参数、ROI 参数、状态转换参数等。

- `templates/`  
  数字模板图像，用于后续的模板匹配与目标跟踪。

## 四、代码整体思路

本项目整体采用**传统图像处理 + 状态机控制**的思路实现，不依赖深度学习方法。

### 1. 锥桶检测与道路分析
程序首先从摄像头图像中提取颜色区域，对锥桶相关区域进行分割，并结合形态学处理、轮廓分析等方法获取道路与障碍物信息。

核心思路包括：
- 颜色空间转换与阈值分割；
- 形态学开闭运算，减少噪声并增强连通区域；
- 轮廓提取与区域筛选；
- 基于图像下半部分区域信息估计可行驶方向。

### 2. 状态机控制道路行驶过程
为了实现较稳定的任务执行，程序将整套流程划分为多个状态，例如：
- 初始循迹；
- 接近障碍物；
- 盲区保持；
- 右转进入空隙；
- 通过弯道；
- 驶出道路；
- 搜索目标；
- 跟踪目标。

这种设计方式的优点是：
- 逻辑清晰；
- 便于调试；
- 便于在不同阶段采用不同控制策略；
- 更适合课程设计场景下的任务分解与展示。

### 3. 自动切换到数字目标跟踪
在完成道路行驶并驶出锥桶区域后，程序会根据状态条件自动切换到目标搜索与跟踪阶段，而不是通过人工操作完成模式切换。

这部分主要包括：
- 自动判定道路阶段结束；
- 进入目标搜索状态；
- 使用模板匹配识别数字目标；
- 根据目标位置与大小调整小车速度和转向，实现持续跟踪。

### 4. 目标跟踪方法
数字目标跟踪主要采用模板匹配思路实现，结合多尺度匹配与简单比例控制完成位置修正和前进控制。

其优点在于：
- 实现相对直接；
- 便于与课程中的图像处理知识对应；
- 不依赖深度学习训练过程；
- 适合作为课程设计中的传统视觉方法实现方案。

## 五、项目特点

本项目具有以下特点：

- 基于 **ROS + OpenCV** 实现；

- 采用**状态机**管理复杂任务流程；
- 实现了从道路行驶到目标跟踪的**自动切换**；

- 符合课程设计中对“功能完成度、自动化程度、稳定性和可解释性”的要求。
